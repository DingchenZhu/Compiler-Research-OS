# Research Idea Workflow

## Purpose

Use this for managing ideas from raw observation to paper or artifact.

## Pipeline

```text
Observation
-> Problem
-> Hypothesis
-> Related Work
-> Experiment Design
-> Prototype
-> Result
-> Analysis
-> Limitation
-> Paper Claim
-> Paper Draft
-> Artifact
-> Open Source
```

## Maturity Scale

```text
0 = raw note
1 = seed idea
2 = problem defined
3 = hypothesis exists
4 = experiment designed
5 = evidence exists
6 = paper claim candidate
```

## Promotion Rules

Promote an idea only when evidence improves:

- seed -> problem: clear pain point and scope
- problem -> hypothesis: testable statement
- hypothesis -> experiment: baseline, metrics, variables defined
- experiment -> claim: evidence exists and limitations are written

## Anti-Patterns

- Turning every learning task into a research idea.
- Writing claims before baseline.
- Ignoring failed experiments.
- Creating broad ideas with no minimal experiment.

## Required Links

Every research idea should link to at least one:

- source note
- paper note
- experiment
- knowledge node
- architecture decision
